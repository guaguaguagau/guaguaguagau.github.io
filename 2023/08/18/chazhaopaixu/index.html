折半查找（二分查找）

  1. 适用场合：有序顺序表的查找

  2. 算法思想：

    ​ 先确定待查记录的所在范围，然后逐步缩小范围直到找到或找不到记录为止

    ​ 将表分解为3个查找区间：前后两个区间的规模大致相等，中间区间长度为1。将待查关键字key和中间的R[mid]比较：

    ​ 设指针low和high表示待查元素所在范围的下界和上界，指针mid指示区域中间位置

    if(key == R[mid]) ：查找成功，返回mid
    else if (key<R[mid]) ：前半区继续查找:[low,mid-1]
    else :在后半区继续查找: [mid+1, high]

  3. 性能分析——判定树：

    T(n) = O(logn)

  4. 算法特点

    算法简单，算法性能高

    只适用于关键字有序的顺序表

    只适用于顺序存储，不适用于链式存储

二叉排序树/二叉查找树

  1. 什么是二叉排序树

    ​ 二叉排序树，或者是一颗空树，或者是具有以下性质的二叉树

    （1）若它的左子树不空，则左子树上所有结点的值均小于它的根节点的值

    （2）若它的右子树不空，则右子树上所有结点的值均大于它的根节点的值

    （3）它的左右子树也分别为二叉排序树

  2. 中序遍历二叉排序树： 中序遍历二叉排序树可得到一个关键字的有序序列（递增）

平衡二叉树（AVL）

​ 建立形态均匀的二叉排序树：让二叉排序树左右子树差不多高

  1. 什么是平衡二叉树：

    结点的平衡因子：定义为该结点的左子树的深度减去它右子树的深度

    平衡二叉树：所有结点的平衡因子绝对值≤1的二叉排序树，即二叉排序树中每个结点的平衡因子只可能为：-1、0、1。

  2. 平衡二叉树的构造

    （1）按照二叉排序树的规则进行结点的插入，新插入的结点平衡因子为0；

    （2）每插入一个结点后，从插入位置开始往上，重新计算其祖先的平衡因子；

    （3）当某个结点“失衡”后，根据旋转法则进行调整。

B-树（m叉平衡排序树）

​ B-树是一种平衡的多路查找树，它在文件系统中很有用。

一棵m阶的B-树：或为空树，或为满足下列条件的m叉树

  • 树中每个结点最多有m棵子树、m-1个关键字
  • 若根节点不是叶子结点，则至少有两棵子树
  • 每个结点至少有┌m/2┐棵子树、 ┌m/2┐ -1个关键字
  • 结点内的关键字有序排列;
  • 所有叶子结点都出现在同一层次上
B+树

​ B+树是应文件系统所需而出的一种B-树的变型树。一颗m阶的B+树和m阶的B-树的差异在于

  • 有n棵子树的结点中含有n个关键字
  • 所有叶子结点包含了安全部关键字信息，以及指向这些关键字记录的指针，且叶子结点本身依关键字的大小自小而大顺序链接
  • 所有的非终端结点可以看成索引部分，结点中仅含有其子树中的最大关键字
键树

​ 键树又称数字查找树，树中的每个结点中不是包含一个或几个关键字，而是只含有组成关键字的符号。例如，若关键字是数值，则结点中只包含一个数位；若关键字是单词，则结点中只包含 一个字母字符。

哈希（查找）表

  • 理想的查找情况：不进行比较，根据关键字值直接计算其存储位置。
  • 根据哈希函数和解决冲突的方法，将一组关键字映射到一个连续的有限地址区间上，得到的表称为哈希（查找）表，每个关键字对应的存储位置称哈希地址或散列地址
冲突和同义词

​ 具有相同函数值的关键字对该哈希函数来说称作同义词，这种现象称之为冲突

构造哈希表

  1. 选取好的哈希函数

    ​ 根据元素集合的特性构造均匀（哈希地址取值是等概率的）,即若对于关键字集合中的仁义关键字，经哈希函数映像到地址集合中任何一个地址的概率是相等的

    （1）直接定址法：取关键字或关键字的某个线性函数值为哈希地址

    （2）数字分析法：取关键字中某些取值分布较均匀的数位作为哈希地址

    （3）折叠法：将关键字分割成位数相同的若干部分（最后一部分的位数可以较少），并取各部分的叠加和（舍去进位）作为哈希函数。

    （4）平方取中法：取关键字的平方，然后根据哈希表地址区 间长度m的大小，选取平方数的中间若干位作为哈希地 址。

概述

  • 排序是将一组“无序”的记录序列调整为“有序”的记录序列。

  • 排序操作的基本操作：“比较”、“移位”。

  • 排序算法的稳定性：排序后两个记录的前后顺序不改变，称排序算法是稳定的算法，否则是不稳定的算法。

  • 排序分类：排序后两个记录的前后顺序不改变，称排序算法是稳定的算法，否则是不稳定的算法。
